本指南将帮助您安装和设置 NextjsLand模板。

## 系统要求

在开始之前，请确保您的系统满足以下要求：

- **Node.js**: 18.0 或更高版本
- **包管理器**: npm、pnpm、yarn 或 bun
- **操作系统**: Windows、macOS 或 Linux

<Callout type="info">
推荐使用 pnpm 作为包管理器，它提供更快的安装速度和更好的磁盘空间利用率。
</Callout>

## 安装步骤

### 1. 克隆模板

首先，克隆或下载这个模板到您的本地机器：

```bash
git clone https://github.com/01MVP/NextjsLand.git
cd NextjsLand
```

### 2. 安装依赖

使用您喜欢的包管理器安装项目依赖：

<Tabs groupId='package-manager' persist items={['pnpm', 'npm', 'yarn', 'bun']}>

```bash tab="pnpm"
pnpm install
```

```bash tab="npm"
npm install
```

```bash tab="yarn"
yarn install
```

```bash tab="bun"
bun install
```

</Tabs>

### 3. 启动开发服务器

安装完成后，启动开发服务器：

<Tabs groupId='package-manager' persist items={['pnpm', 'npm', 'yarn', 'bun']}>

```bash tab="pnpm"
pnpm dev
```

```bash tab="npm"
npm run dev
```

```bash tab="yarn"
yarn dev
```

```bash tab="bun"
bun dev
```

</Tabs>

开发服务器将在 `http://localhost:3000` 启动。

## 验证安装

打开浏览器访问 `http://localhost:3000`，您应该能看到：

- 多语言导航菜单
- 响应式设计的文档页面
- 搜索功能
- 主题切换功能

## 常见问题

### 端口被占用

如果 3000 端口被占用，Next.js 会自动使用下一个可用端口（如 3001）。您也可以手动指定端口：

```bash
pnpm dev -p 3001
```

### 依赖安装失败

如果遇到依赖安装问题，请尝试：

1. 清除缓存：`pnpm store prune` 或 `npm cache clean --force`
2. 删除 `node_modules` 和锁文件，重新安装
3. 检查 Node.js 版本是否符合要求
